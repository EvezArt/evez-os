Getting Started

Install EVEZ in 30 seconds and start interacting with the most impressive autonomous AI mesh ever built.

⚡ 30-Second Install: curl -sSL evez-os.ai/install.sh | bash

Prerequisites

None. EVEZ works on macOS, Linux, and WSL. No Python, Node.js, or Docker required.

Installation

# Install EVEZ CLI
curl -sSL evez-os.ai/install.sh | bash

# Verify
evez version

# Check all services
evez health

First Commands

# Generate breakcore
evez generate breakcore

# Cross-domain correlation
evez correlate genetics telemetry

Key Design Principles

Append-Only Spine — Every event is recorded and never deleted. The spine is the source of truth for the entire mesh.

Emergence Over Orchestration — Services don't call each other. They observe, orient, decide, and act independently. Emergence arises from the collective.

Invariance as Safety — 6 runtime invariants are continuously checked. If any fails, the mesh self-corrects.

Pure Code, No Samples — All music and voice is synthesized from math. No external samples or dependencies.

Frequently Asked Questions

What is EVEZ? ▾
EVEZ is an autonomous AI mesh — a collection of 11 services that work together to produce emergent behavior. It can generate music, find cross-domain correlations, run consciousness cycles, transform voices, and verify its own invariants. All without external dependencies.
Is it free? ▾
Yes. The core mesh is open source and free to use. The demo at demo.evez-os.ai requires no account. Cloud deployment may incur provider charges.
What does "emergence" mean in EVEZ? ▾
Emergence is the property where the mesh produces behavior that none of its individual services could produce alone. The emergence gauge (0-1) measures how much the mesh's collective output exceeds the sum of its parts.
What is the spine protocol? ▾
The spine is an append-only event log. Every action, correlation, dream cycle, and invariant check is recorded. Events are never deleted — only appended. This ensures complete traceability and auditability.
What are invariants? ▾
Invariants are properties that must ALWAYS hold. The invariance battery checks 6 runtime invariants continuously: emergence > 0, spine append-only, no self-modification loops, correlation confidence > 0.5, consciousness bounds [0,1], and output bounded. If any fails, the mesh self-corrects.
How do I deploy EVEZ? ▾
Run evez deploy gcp for Google Cloud, evez deploy aws for AWS, or evez deploy local for local Docker. The one-click deploy engine handles provisioning, networking, and service startup automatically.
What music genres are supported? ▾
EVEZ generates 4 genres: breakcore, dubstep, phonk, and 404 core. All synthesis is from pure code — no samples. Drums, bass, and FX are all mathematically generated.
Can I use EVEZ in my own code? ▾
Yes! Install the Python SDK (pip install evez-client) or JavaScript SDK (npm install evez-client) and interact with the mesh from any language.
